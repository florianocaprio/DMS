## COSA COSTRUIRE

### 1. BACKEND — sostituire routes/bootstrap.ts

Mantieni GET /bootstrap/status invariato (serve a BootstrapGuard).

Sostituisci POST /bootstrap/admin con un nuovo endpoint:

`POST /bootstrap/complete`

Pubblico (prima di requireAuth), attivo SOLO se GET /bootstrap/status
ritorna { needed: true }. Se il sistema è già configurato risponde 409.

Payload JSON:

```json
{
  "admin": {
    "area": "string|null",
    "section": "string|null",
    "telefono": "string|null"
  },
  "organizzazione": {
    "nome": "string (required)",
    "tipologia": "associazione|parrocchia|azienda|cooperativa|comune|altro",
    "email": "string|null",
    "telefono": "string|null",
    "indirizzo": "string|null",
    "citta": "string|null"
  },
  "protocollo": {
    "prefisso": "string (required)",
    "anno": "number (required, es. 2026)",
    "descrizione": "string|null"
  }
}
```

Il backend esegue tutto in una singola transazione Drizzle:

1. Verifica che isBootstrapMode() sia true (conteggio users = 0),
   altrimenti 409.

2. Legge l'identità dell'utente dal token Clerk (getAuth(req).userId),
   poi chiama clerkClient.users.getUser() per ottenere email e nome.
   Se non c'è sessione Clerk attiva risponde 401.

3. Crea il primo utente nella tabella users:
   - clerkUserId dal token
   - email e name da Clerk
   - role: "admin"
   - area e section dal payload admin (possono essere null)
   - lastLoginAt: now()

4. Salva i dati organizzazione in app_settings con chiavi:
   - org.nome
   - org.tipologia
   - org.email
   - org.telefono
   - org.indirizzo
   - org.citta
   - setup.completedAt (timestamp ISO)

5. Salva i dati del protocollo in app_settings con chiavi:
   - protocollo.prefisso
   - protocollo.anno
   - protocollo.descrizione

6. Assicura che esista il fascicolo di default "Archivio Documenti"
   chiamando la funzione ensureDefaultDossier() già presente in
   lib/ensureDefaults.ts.

Risposta successo: 201 con { ok: true }.

Aggiorna isBootstrapAllowedRequest in clerkAuth.ts per includere
POST /bootstrap/complete nella whitelist pubblica (senza auth).
Rimuovi POST /bootstrap/admin dalla whitelist se non serve più.

### 2. FRONTEND — wizard a 3 step

Sostituisci completamente artifacts/dms/src/pages/bootstrap.tsx
con un wizard a 3 step (non 4 come nel Magazzino Solidale, perché
non c'è lo step credenziali — Clerk ci pensa già).

Il wizard usa SOLO stato React locale. Nessuna chiamata API intermedia.
L'unica chiamata è la POST finale al completamento dello step 3.

Layout generale:
- Sfondo: bg-background min-h-[100dvh]
- Card centrata: max-w-lg w-full, rounded-2xl, border, shadow-sm
- In cima: logo AIM (già importato in App.tsx come logoUrl), titolo
  "Configurazione iniziale", badge "Primo avvio" in amber
- Indicatore step: "Passo N di 3" + barra di progresso (usa il
  componente Progress di shadcn/ui)
- Bottoni "Indietro" / "Avanti" / "Avvia sistema" in fondo

---

#### STEP 1 — Organizzazione

Titolo: "La tua organizzazione"
Sottotitolo: "Questi dati verranno usati per intestare protocolli e documenti."

Campi:
- Nome organizzazione* (required)
  placeholder: "es. Parrocchia San Giovanni / ASD Angeli in Moto"
- Tipologia* (required) — Select:
  - associazione → "Associazione di volontariato / APS"
  - parrocchia → "Parrocchia / Ente religioso"
  - azienda → "Azienda / Cooperativa sociale"
  - comune → "Comune / Ente pubblico"
  - altro → "Altro"
- Email istituzionale (opzionale)
- Telefono (opzionale)
- Indirizzo sede (opzionale)
- Città (opzionale)

Validazione "Avanti": nome e tipologia obbligatori.

---

#### STEP 2 — Impostazioni protocollo

Titolo: "Protocollo documenti"
Sottotitolo: "Definisce come verranno numerati i protocolli in entrata e uscita."

Banner informativo (blue, icona FileText):
"Il prefisso identifica la tua organizzazione nei numeri di protocollo.
Esempio: con prefisso 'AIM' i protocolli saranno AIM-2026-E-000001."

Campi:
- Prefisso protocollo* (required, max 6 caratteri, uppercase automatico)
  placeholder: "es. AIM"
  note sotto: "Da 2 a 6 lettere, senza spazi. Non modificabile dopo l'avvio."
- Anno di avvio* (required, default: anno corrente)
  note sotto: "Anno da cui parte la numerazione dei protocolli."
- Descrizione registro (opzionale)
  placeholder: "es. Registro protocolli Angeli in Moto"

Validazione "Avanti":
- prefisso obbligatorio, solo lettere A-Z, lunghezza 2-6
- anno obbligatorio, intero >= 2020

---

#### STEP 3 — Profilo amministratore + Riepilogo

Titolo: "Il tuo profilo e riepilogo"
Sottotitolo: "Completa il tuo profilo amministratore e verifica i dati."

Sezione profilo (prima):
Banner che mostra l'identità Google già autenticata (read-only):
- Avatar (useUser() di Clerk oppure avatarUrl), nome, email
  con label "Account Google — verificato" e badge verde

Campi aggiuntivi del profilo (editabili):
- Area / Ufficio (opzionale) — es. "Segreteria", "Direzione"
- Sezione (opzionale) — es. "Front Office"
- Cellulare (opzionale)

Separatore visivo poi sezione riepilogo (read-only):

Riepilogo "Organizzazione":
- Nome, Tipologia (label leggibile), Email, Telefono, Città

Riepilogo "Protocollo":
- Prefisso, Anno, Descrizione
- Esempio numero generato: "{PREFISSO}-{ANNO}-E-000001" in stile codice (font-mono)

Bottone: "Avvia sistema" (primary, full width)
- Durante pending: spinner + "Configurazione in corso..."
- In caso di errore: alert rosso sotto con messaggio ricevuto dal server

#### Dopo il successo

Quando POST /bootstrap/complete risponde 201:
1. Invalida la query di bootstrap status nel QueryClient TanStack:
   queryClient.invalidateQueries({ queryKey: ["bootstrap", "status"] })
   oppure chiama onComplete() che è già la prop passata da BootstrapGuard
   in App.tsx — questo fa uscire automaticamente dal wizard e mostra l'app.
2. Mostra un toast: "Sistema configurato. Benvenuto in ProtocolloDigitale."
